# CLAUDE.md — Настройка среды разработки

## Проект

**PhotoReportGenerator** — консольное приложение для генерации фотоотчётов в формате `.docx`.
Язык: Python 3.12+. Запуск: `python main.py`.

## Структура

```
Photo_in_word/
├── main.py              # точка входа, CLI + интерактивный режим
├── src/
│   ├── config.py        # DocumentConfig, валидация, константы
│   ├── document_builder.py  # сборка .docx
│   ├── image_loader.py  # загрузка и классификация изображений
│   └── utils.py         # вспомогательные функции
├── requirements.txt
├── CLAUDE.md            # этот файл
└── telegram_bot_rules.md  # правила разработки Telegram-ботов
```

## Команды

```bash
# Установка зависимостей
pip install -r requirements.txt

# Запуск (интерактивный режим)
python main.py

# Запуск с аргументами
python main.py --path "C:\Фото" --count 4

# Тесты
pytest tests/ -v

# Линтер
ruff check .
```

## Рабочий процесс (Claude Code Best Practices)

- **Всегда начинать с plan mode** перед реализацией
- Коммитить после каждой завершённой задачи
- Не превышать 50% контекста до ручного compacting
- CLAUDE.md не должен превышать 150 строк
- Использовать wildcard-синтаксис в permissions вместо skip-permissions

## Стандарты кода

- Типизация обязательна для всех функций (включая `-> None`)
- Docstrings по PEP 257 для всех публичных функций и классов
- Линтер: `ruff`
- Тесты: `pytest` (не unittest), каталог `tests/`
- Настройки через переменные окружения (`.env` файл), не хардкод

## Настройки ИИ-помощника

- При неясных требованиях — сначала уточнить, потом реализовывать
- Не добавлять лишний код сверх запрошенного
- Не рефакторить незатронутый код
- Не добавлять docstrings/комментарии к коду, который не менялся
- Предпочитать edit существующих файлов созданию новых
