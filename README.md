# PhotoReportGenerator

Консольное приложение для автоматизированного создания фотоотчётов в формате MS Word (.docx). Обрабатывает папку с изображениями, классифицирует их по ориентации (альбомная/книжная) и формирует документ с фотографиями в сетке на страницах А4.

## Возможности

- Поддержка форматов: JPG, JPEG, PNG, WebP, BMP, GIF, TIFF, HEIC (iPhone)
- Строгое разделение по ориентации: на одной странице только альбомные или только книжные фото
- Сетка на странице (1–12 фото на лист), невидимая таблица, отступы между фото
- Автоматическое имя файла: `PhotoReport_YYYY-MM-DD_HH-MM.docx`
- Запуск из консоли без установки приложения (достаточно Python и зависимостей)

## Установка зависимостей

```bash
pip install -r requirements.txt
```

## Запуск

**Интерактивный режим** (программа запросит путь, количество фото на страницу и отступы):

```bash
python main.py
```

**С аргументами командной строки:**

```bash
python main.py --path "C:\ПапкаСФото" --count 4
```

```bash
python main.py -p "C:\Фото" -c 2 -o "C:\Отчёты"
```

Сохранить в конкретный файл:

```bash
python main.py -p "C:\Фото" -c 4 -o "C:\Отчёты\МойОтчёт.docx"
```

| Аргумент | Короткий | Описание |
|----------|----------|----------|
| `--path` | `-p` | Путь к папке с фотографиями |
| `--count` | `-c` | Количество фото на одной странице (целое число > 0) |
| `--output` | `-o` | Путь для сохранения: каталог или полный путь к файлу .docx (необязательно) |

По умолчанию документ сохраняется в папку с фотографиями. При указании `--output` в качестве каталога файл сохраняется в этот каталог с автоматическим именем по шаблону.

## Возможные ошибки и решения

| Ситуация | Сообщение | Решение |
|----------|-----------|---------|
| Несуществующий или неверный путь к папке | Папка не найдена | Проверьте путь, используйте кавычки при пробелах |
| В папке нет поддерживаемых изображений | В указанной папке нет поддерживаемых изображений | Добавьте файлы .jpg, .png и т.п. в указанную папку |
| Нет прав на запись | Нет прав на запись в «…» | Запустите от имени администратора или укажите другую папку через `--output` |
| Повреждённый файл с расширением изображения | В логе: пропуск повреждённого файла | Файл пропускается, работа продолжается; при необходимости удалите или замените файл |

## Сборка в .exe

При необходимости можно собрать исполняемый файл (без установки Python на целевом компьютере):

```bash
build_exe.bat
```

Будет создан `dist\PhotoInWord.exe`.

## Требования

- Python 3.12 или 3.13
- Зависимости из `requirements.txt` (python-docx, Pillow и др.)
